  • replaceVariables(object: any, replacements?: any): any

  • Inherited from ConfigApplication.replaceVariables

    Allow variable inside object strings

    Example

    replaceVariables({
 myobj: "${test.replace}"
}, {
 test: {
   replace: 'plop'
 }
})

    will return

    {
 myobj: 'plop'
}

    By default the replacements map contains

    {
 git: GitInformation,
 package: 'package.json content',
 deployment: string,
 now: number,
 ...replacements
}

    See: GitInformation

    Parameters

      a duplicated object with replacement done
